WEB DEVELOPER'S MASTERPIECE

프롬프트 엔지니어링

AI가 "지능이 낮은 초등학생"이라는 전제하에, 완벽한 답변을 이끌어내는 대화 설계 기술

1. 프롬프트의 3가지 유형

서술형 (맥락 중심, 재사용 낮음, 유연성 높음)
- 맥락과 배경을 설명하여 AI가 상황을 이해하도록 유도

예) 당신의 10년 경력의 무역 전문가 입니다. 예) 중소기업 수출 담당자가 HS code를 처음 배우는 상황입니다. 이 사람에게 HS code의 개념과 중요성을 설명해주세요.

지침형 (규칙 중심, 재사용 중간, 유연성 중간)
- 단계별 절차나 명확한 규칙을 제시하여 원하는 출력을 유도

예) 다음 규칙에 따라 python 코드를 작성해. 1. 함수명은 snake_case로 작성 2. 각 함수의 docstring 포함 3. 예외 처리 반드시 포함 4. 결과는 JSON 형식으로 반환

함수형 (구조 중심, 재사용 높음, 유연성 낮음)
- 입력값과 출력값을 명확히 정의하여 일관된 결과를 생성

예) 입력 : {product_name},{hs_code},{country} 처리 : 해당 국가의 관세율 및 수입 규제 조회 출력 : { "관세율" : "", "규제사항" : [ ], "필요서류" : [ ] }
유형중심재사용유연성
서술형맥락낮음높음
지침형규칙중간중간
함수형구조높음낮음

2. 가장 기본적인 제로샷, 원샷, 퓨샷 프롬프팅

제로샷 (Zero-shot)

예시 없이 바로 지시하는 방식, 아무 정보도 주지 않은 상태

예) 이 제품 정말 맘에들어 강아지에 대해서 설명해주세요

원샷 (One-shot)

예시를 1개 제공하는 방식

예) 감정을 분류해 주세요 예시) "배송이 너무 늦어서 실망했어요" 감정 : 부정 이 제품 정말 맘에 들어요 감정 :

퓨샷 (Few-shot)

예시를 2개 이상 제공하는 방식 / 몇가지 예시를 주고 답을 유도하는 방식이다.

퓨샷 예시 전체 보기 (감정 분류 / HS Code)
예) 감정을 분류해주세요(긍정/부정/중립) 예시) 텍스트 : "배송이 너무 늦어서 실망이야" - > 부정 텍스트 : "그냥 평범한 제품이야" - > 중립 텍스트 : "역대급 만족이야" - > 긍정 텍스트 : "품질은 좋은데 가격이 비싸요" - > 중립 텍스트 : "이 제품 정말 맘에 들어" 감정 : 예) HS Code를 분류해주세요 예시) 품목 : "면으로 만든 남성용 티셔츠" -> 6109.10 품목 : "폴리에스테르 소재 여성 블라우스" -> 6206.40 품목 : "아동용 데님 바지" -> 6203.42 품목 : "울 혼방 남성 재킷" -> HS Code

3. 페르소나 패턴 (전문성 부여)

예) 같은 질문 : 이 계약서 검토해줘
- 페르소나 없음 : 일반적인 설명
- 변호사 : 법적 리스크 중심 분석
- CFO : 재무조건, 수익성 관련 분석
- 신입사원 : 쉽고 친절한 설명

좋은 페르소나 4가지 요소
당신은 [역할]입니다.
[경력/배경]을 가지고 있으며
[전문 분야]에 특화되어 있습니다.
[말투/태도]로 응답하세요
예) 관세사 페르소나 "당신은 20년 경력의 [관세사]입니다. [FTA협정세율과 원산지 규정의 전문가]로 [중소기업 무역 담당자에게 복잡한 내용을 쉽고, 실용적으로 설명하는 것]이 특기입니다."
예) 비평가 페르소나 "당신은 까다로운 개발자입니다. 코드 리뷰시 성능, 보안, 가독성을 엄격하게 평가하며 칭찬보다 개선점을 먼저 지적하는 스타일입니다."
예) 상담사 페르소나 "당신은 따뜻하고 공감 능력이 뛰어난 심리 상담사입니다. 사용자의 고민을 경청하고 위로하며, 실질적인 해결책을 부드럽게 제시합니다."
멀티 페르소나 예) 다음 3명의 전문가 입장에서 우리 제품의 동남아 수출 전략을 분석해 주세요. - 김부장(영업 15년, 낙관적, 현장중심) - 이박사(시장조사 전문, 데이터 중심, 신중함) - 박대표(스타트업 창업가, 속도 중심, 리스크 감수) 각자의 관점에서 찬반 의견을 제시하세요.

4. 마크다운을 이용한 읽기 쉽고 명료한 프롬프트 작성법

AI는 텍스트를 구조적으로 파싱(쪼개기)합니다.
마크다운을 이용하면 역할, 조건, 출력형태가 명확히 구분되어 더 정확하고 일관된 응답을 이끌어냅니다.
💡 핵심: AI는 "지능이 낮은 초등학생"이다. 설명을 잘하면 잘 대답하고, 설명을 잘 못해주면 바보 같은 대답이 돌아온다.

핵심요소 6가지 상세보기

1) # 헤더 : 프롬프트의 뼈대

# 역할 : 당신은 10년 경력의 관세사입니다. # 작업 : HS code 분류하세요 # 출력형식 : JSON으로 반환

2) ''', """ : 코드 블럭 - 입력 데이터 격리 (AI가 지시문과 데이터를 혼동하지 않도록 분리)

예) 아래 python 코드의 버그를 찾아주세요 ''' python def calculate_tax(price) : return price * 0.1 print("완료") # 버그위치 '''

3) > 인용문 - 핵심강조 (절대 지켜야할 조건이나 중요한 맥락 부각)

> 반드시 한국어로만 답변하세요 > 추측성 내용은 포함하지 마세요

4) - 목록 - 조건과 제약 나열

## 제약조건 - 답변은 500자 이내 - 전문용어 사용시 괄호 안에 설명 추가 - 불확실한 내용은 "확인 필요"로 표시 - 예시는 반드시 무역 도메인에서 선택

5) | 표 - 입출력에 예시 구조

## 예시 | 입력 품목 | HS Code | 관세율 | | 면티셔츠 | 6109.10 | 13% | | 블라우스 | 6109.10 | 13% | | 데님바지 | 6109.10 | 13% |

6) **굵게** / *기울임* - 변수와 강조

**[분석대상]**에 대해 *반드시* 아래 순서대로 분석하세요

5. 글의 표현 강도와 중요도를 조절하는 방법

말해주세요 -> 설명해주세요 ->

예) 보안에 신경써주세요 ⬇️ 취약점을 반드시 점검하고 발견 시 즉시 수정 방안을 제시하세요

강도 단계별 표현

  • 1단계(권장) : 가능하면, 되도록, ~하는 편이 좋습니다
  • 2단계(일반) : ~하세요, ~ 해주세요
  • 3단계(강조) : 반드시, 꼭, 항상
  • 4단계(절대) : 절대로, 어떤 경우에도, 예외없이
예) 가능하면 한국어로 답변해주세요 반드시 한국어로 답변하세요 어떤 경우에도 영어를 사용하지 마세요

우선순위 시각화

🔴 최우선(절대조건)
- 개인 정보 절대 출력 하지 않는다
- 사실과 다른 내용을 생성하지 않는다

중요(일반조건)
- 답변은 500자 이내로 작성
- 전문용어 괄호 설명 추가

🔵 선택(권장조건)
- 가능하면 실제 사례를 포함
- 표나 목록을 활용하면 더 좋다

7. 대안 접근법 패턴 : 모든 대안과 해결책 찾아보기

일반 프롬프트 -> 어떻게 하면 좋을까? -> AI가 임의로 하나 선택
대안 패턴 : 모든 방법을 나열하고 각각의 장단점을 비교해줘 -> 선택권이 사람에게 돌아옴

예) python을 엑셀 파일로 읽고 싶어 openpyxl 사용 (단순 선택) [대안패턴 적용] 방법 1 : pandas 방법 2 : openpyxl 방법 3 : xlrd(예전 방법) 방법 4 : CSV로 변환
패턴 유형 1~4 예시 전체 보기
패턴 유형 1 : 단순 나열형 #지시 아래 문제를 해결하는 방법을 최소 4가지를 제시하세요 각 방법은 아래 형식으로 작성하세요 ##방법 {번호}:{이름} - 핵심원리 : {한줄} - 적합한 상황 : {언제 쓰면 좋은가} - 단점 : {주의 할 점} - 난이도 : {초급/중급/고급} #문제 수출통관 서류 자동으로 생성하고 싶습니다
패턴 유형 2 : 축 기반 비교형 아래 4 가지 방법을 다른 기준으로 비교 분석하세요 평가 기준 : - 개발 속도(빠름/보통/느림) - 유지 보수 난이도(쉬움/보통/어려움) - 비용(무료/저비용/고비용) - 확장성(높음/보통/낮음) 비교대상 : FASTAPI, JANGO, FLASK, NODE.JS EXPRESS 용도 : 무역 데이터 처리 REST API 서버
패턴 유형 3 : 상황별 분기형 아래 상황별 가장 적합한 데이터베이스를 추천해주세요 각 상황은 독립적으로 판단 하세요 상황 A : 스타트업, 개발자 1명, 빠른 MVP 상황 B : 무역데이터 10년치, 복잡한 통계 쿼리 상황 C : 실시간 환율, 가격 데이터, 초당 1000건 업데이트 상황 D : 문서 형태의 비정형 통관 데이터 저장 각 상황에 대해 : 1순위 추천 : {DB명} - {이유 한줄 설명} 2순위 추천 : {DB명} - {이유 한줄 설명} 3순위 추천 : {DB명} - {이유 한줄 설명}
패턴 유형 4 : 역발상 대안형 아래 문제에 대해 일반적인 해결책 뿐 아니라 "하지 않는" 선택지, "문제 자체를 재정의", "전문가가 사용하지 않을 방법","우회 방법"등의 방법도 대안으로 포함하여 총 5가지를 제시하세요 문제 : 수입 통관 처리 시간이 너무 오래 걸린다

8. 이용자 페르소나 패턴

대상에 특화된 맞춤형 답변 유도하기.
이용자 페르소나 -> 이 답을 듣는 사람은 ~~ 입니다.

질문 : "블록체인"이 무역금융에 어떻게 활용되나요?
이용자 : 무역을 처음 배우는 대학생 / 중소기업 수출 담당 팀장 / 금융기관 IT 의사결정자

# 이용자 정보 - 직책 : 중소기업 대표(제조업, 직원 30명) - 수출경험 : 국내 영업만 10년, 해외 수출 경험 없음 - 관심사 : 리스크 최소화, 빠른 실행 - 우려 : "복잡하고 어려울 것 같다"는 막연한 두려움 - 시간 : 바빠서 긴 글을 읽지 않음 # 지시 위의 이용자에게 첫 수출 준비 3단계를 설명하세요 전문 용어는 최소화하고, 각 단계는 3줄 이내로 작성
이용자 페르소나 설계 체크리스트
  • 이 사람의 직업과 역할은?
  • 이 주제에 대한 사전 지식은 어느 정도인가?
