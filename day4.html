26. 다중 관점 기법 : 여러 관점으로 보기

단일 시점의 분석은 실용성은 높을 수 있으나 핵심적인 리스크나 감성적인 요소를 놓치기 쉽습니다. 다중 관점은 입체적인 해결책을 만드는 가장 빠른 방법입니다.

전문가 관점

높은 실용성, 전문성
(감성 결여 가능성)

사용자 관점

높은 공감, 사용자 경험
(전문성 결여 가능성)

낙관론

가능성과 기회 발견
(리스크 간과 가능성)

비관론

리스크와 한계 식별
(기회 상실 가능성)

다중 관점이 만드는 시너지

  • 충돌하는 지점 : 진짜 숨겨진 문제 발견
  • 동의하는 지점 : 변하지 않는 확실한 진실 발견
  • 보완하는 지점 : 결점 없는 완전한 해결책 발견

다중 관점의 5가지 원칙

  • 이해 관계자 차원 : 관련된 사람들의 서로 다른 입장
  • 전문 분야 차원 : 다른 학문과 직종의 시각
  • 시간 차원 : 과거, 현재, 미래의 시각
  • 가치관 차원 : 다른 철학과 우선순위
  • 문화, 맥락 차원 : 다른 환경 배경의 시각

# 방법 1 : 병렬 관점 소환

아래 주제를 {N가지 관점}에서 동시에 분석하세요.
각 관점은 독립적으로 분석하되 마지막에 관점들이 만나는 교차점을 찾아주세요.

# 방법 2 : 대화 시뮬레이션

{관점A}, {관점B}, {관점C}가 같은 테이블에 앉아 이 주제를 토론합니다.
각자의 주장, 반론, 그리고 결론을 써주세요.

# 방법 3 : 레이어 분석

같은 주제를 {레이어1}에서 {레이어 N}까지 겹겹이 쌓아가며 분석하세요.
아래 층위가 위 층위를 어떻게 규정하는지 보여 주세요.

27. 심사숙고 유도 기법 : 깊이 생각하고 답변하게 하기

AI가 즉각적인 답변을 내놓기보다 중간 과정을 거치게 하여 논리적 완성도를 높이는 기법입니다.

심사숙고 메커니즘

  1. 사고 공간 확보 : 답을 쓰기 전에 먼저 생각할 시간을 부여
  2. 사고 단계 강제 : 건너뛸 수 없는 논리적 중간 단계 설계
  3. 자기 검증 루프 : 최종 답변 전 스스로 오류를 검토하게 함
  4. 불확실성 명시 : 아는 것과 모르는 것을 명확히 구분

❌ 일반적인 입력 질문
"우리 회사가 베트남에 진출해야 할까?"

⭕ CoT(Chain of Thought) 적용 프롬프트
"우리 회사가 베트남에 진출해야 할지 단계별로 생각하면서 판단해줘. 각 단계에서 내린 중간 결론을 보여줘."

28. APE(Automatic Prompt Engineer) : 자동 최적화

AI를 사용하여 최적의 프롬프트를 생성하고 평가하는 프로세스입니다.

# APE 실행 순서

Step 1 : 후보 프롬프트 생성
(역할 부여형, 단계별 지시형 등 다양한 전략 사용)

Step 2 : 각 후보 자체 평가
(명확성, 완전성, 실행 가능성, 품질 유도력 기준 점수화)

Step 3 : 상위 3개 선택 후 교차 최적화

Step 4 : 최적 프롬프트 출력
(최적 프롬프트 전문, 채택 이유, 사용 시 주의사항)

29. AI 프롬프트 작성법 : 5가지 열쇠

1. 명확하고 구체적으로 지시하기
ex. 월 예산 50만원으로 40대 타겟 매출을 20% 올릴 아이디어 3개 제안해줘

2. CONTEXT(맥락) 제공
ex. 나는 10년차 재무 상담사야, 불안해하는 50대 직장인에게 따뜻한 조언으로 써줘

3. 말보다 샘플(Example) 보여주기
ex. 내가 쓴 이 예시와 비슷한 톤앤매너로 고객 감사 메일 써줘

4. 거대한 업무 잘게 쪼개기
ex. 먼저 핵심 컨셉 3개만 제안해줘, 그 중 하나로 상세 목차 짜보자

5. AI를 파트너로 대하기
ex. 이 부분은 좀 더 전문가 답게 고쳐줘, 사례를 하나 더 추가해 줘
